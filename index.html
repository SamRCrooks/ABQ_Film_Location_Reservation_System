Development Process - Created by Sam Crooks

Motivation: With news last year of Netflix buying Albuquerque studios and more recently NBC Universal announcing to build its own new studio complex this year, there is no question that the film industry is booming in New Mexico. Film shoots often require numerous locations, and the New Mexico film office’s website contains a database of over 8,000 available locations for production companies and independent filmmakers to reserve. However, the current system is inefficient because those who want to reserve locations must contact the owner of the location of interest directly. Furthermore, there is no way to quickly determine whether or not a site has already been reserved by someone else at a specified time. To address these problems, I developed an interactive map-based reservation web app for film sites in Albuquerque. Below details the process in developing the app.
Tools Used:
1. Data Extraction and Processing
Location data from Albuquerque’s open data website was used [2]. It contains a dataset of movie locations for 1,000 productions shot (406 unique locations) in non-residential locations within the city of Albuquerque that required a film permit. The data is stored as a JSON file (JavaScript Object Notation). Example of location attributes within the JSON file:

2. Plotting Locations on a Map
    A map was created using Leaflet, an open-source JavaScript library for interactive maps [4].
    Downloaded a movie camera icon PNG. Created red and green versions. Green indicates a location is vacant. Red indicates reserved.
    A for loop iterates through the coordinates and site names. A new map marker is created at each coordinate and the corresponding site name is bound each marker’s pop-up.
    When a marker is clicked on, the pop-up becomes visible.
The only necessary elements that I need from this dataset are the site names and their corresponding latitude and longitude coordinates so that they can be plotted on a map. Unfortunately, this dataset doesn’t include latitude and longitude coordinates; however, it does have addresses which can be converted to coordinates later on. In data_conversion.js, I extracted the site names and addresses of each location by creating a for loop which iterates through the location attributes and saves the site names to a list called sites and saves the addresses to list called addresses. The addresses were then converted to a list of coordinates using the MapQuest geocoding tool [3]. Finally, the cleansed address and coordinate data was saved as lists into a new file called data.js. In the future, it may be wise to store the data as a JavaScript object instead of lists because I may need data attributes beyond site names and coordinates, but this is suitable for my purposes.
