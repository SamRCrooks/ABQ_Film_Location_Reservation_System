Development Process - Project created by Sam Crooks

Motivation

There is no question that the film industry is booming in New Mexico, as within the last year Netflix bought Albuquerque studios and NBC Universal announced to build its own new studio complex in Albuquerque. Film shoots often require numerous locations, and the New Mexico film officeâ€™s website provides a database of over 8,000 available locations for production companies and independent filmmakers. However, this current system can be time consuming and inefficient as reservations can only be made by contacting the owner of each location directly. Furthermore, there is no way to quickly determine whether or not a site has already been reserved by someone else at a specified time. To address these limitations, I developed an interactive map-based reservation web app for film sites in Albuquerque. Below details the process in developing the app.

The only necessary elements from this dataset are the site names and their corresponding latitude and longitude coordinates so that they can be plotted on a map. Unfortunately, this dataset doesnâ€™t include latitude and longitude coordinates; however, it does have addresses which can be converted to coordinates. In data_conversion.js, I extracted the site names and addresses of each location by creating a for loop that iterates through the location attributes and then saves the site names to a list called sites and saves the addresses to list called addresses. The addresses were then converted to a list of coordinates using the MapQuest geocoding tool [3]. Finally, the cleansed address and coordinate data was saved as lists into a new file called data.js. In the future, it may be wise to store the data as a JavaScript object instead of lists because I may need data attributes beyond site names and coordinates, but lists are suitable for my purposes.

2. Plotting Locations on a Map
    A map was created using Leaflet, an open-source JavaScript library for interactive maps [4].
    Downloaded a movie camera icon PNG. Created red and green versions. Green indicates a location is vacant. Red indicates reserved.
    A for loop iterates through the coordinates and site names. A new map marker is created at each coordinate and the corresponding site name is bound with each markerâ€™s pop-up.
    When a marker is clicked on, the pop-up becomes visible.


3. Site Reservation Functionality
    Created a green reservation button for each pop-up.
    When the reservation button is clicked, a button action listener is triggered which changes the locationâ€™s icon from green to red.
    Added a calendar widget in each pop up so that the user can select the start and end dates of their reservation.


4. Payment
    A paypal button is prompted after a location is reserved.
    In a final product, the cost would be calculated based on the location and reservation duration.
